symbol是es6新出的属性，特点是：独特且不可变，你真的懂唯一~的定义
es6定制的api：

  • Symbol.iterator：定义对象的默认迭代行为。
  • Symbol.toStringTag：定制对象的 toString() 输出。
  • Symbol.asyncIterator：支持异步迭代。

主要用于依赖注入与私有属性（头疼）

常见应用场景

  1. 私有属性（模拟）

    • 在大型项目中，Symbol 常用于模拟对象的私有属性，使这些属性不会意外暴露或被覆盖。“私有属性”又与私人属性分不开，还得了解不可枚举属性

  2. 依赖注入

    • 在框架或库中，用 Symbol 定义唯一标识符，用于实现服务或依赖注入机制，避免命名冲突。

  3. 扩展第三方库或内置对象

    • 避免破坏原有功能，通过 Symbol 添加新特性：

  4. 枚举成员的唯一标识

    • 在实现枚举类型时，可以用 Symbol 确保每个枚举值都是唯一的：

  5. 元编程与协议扩展

    • 使用 Well-known Symbols 定制和拦截对象行为（如迭代器协议）。

用得多吗？

  • 在普通业务代码中： 使用频率较低，尤其是小型项目或简单应用中，因为其功能更适合特定需求（如依赖注入、扩展库功能）。

  • 在框架与库中：

    使用较为频繁。例如：

    • React 使用 Symbol 来区分不同的 React 元素类型。
    • Redux 用 Symbol 定义私有属性。
    • DI（依赖注入）工具中常用 Symbol 作为唯一标识。
